SERVICE: Auth Service
DESCRIPTION: Manages authentication, token generation (JWT), and security.

OWNED DATA (SCHEMAS):
- Technically accesses 'Users' table (or a dedicated credentials table) to verify passwords.

API ENDPOINTS:
POST /api/v1/auth/login
    - Verifies credentials.
    - Generates Access & Refresh tokens.
POST /api/v1/auth/signup
    - Creates a new user entry (Synchronous call to User-Service or emits UserCreated event).
    - Returns initial tokens.
POST /api/v1/auth/refresh
    - Issues new access token using refresh token.
POST /api/v1/auth/password/reset-request
    - Generates reset token.
    - KAFKA PRODUCER: Emits 'auth.password-reset-requested' (Notification Service listens to send email).
POST /api/v1/auth/password/reset
    - Updates password.

KAFKA EVENTS:
- Produces: `auth.user.logged-in`, `auth.password.reset-requested`
