SERVICE: API Gateway
DESCRIPTION: The entry point for the frontend. Routes requests to specific microservices.

CONFIGURATION:
- Route /api/v1/auth/** -> auth-service
- Route /api/v1/users/** -> user-service
- Route /api/v1/driver/** -> trip-service (mostly), user-service, vehicle-service
- Route /api/v1/lines/** -> route-service
- Route /api/v1/stations/** -> route-service
- Route /api/v1/locations/** -> geolocation-service
- Route /api/v1/routes/** -> trip-service
- Route /api/v1/trips/** -> trip-service
- Route /api/v1/bookings/** -> ticket-service
- Route /api/v1/tickets/** -> ticket-service
- Route /api/v1/payments/** -> payment-service
- Route /api/v1/subscriptions/** -> subscription-service
- Route /api/v1/admin/** -> Splits based on sub-path (e.g., /admin/lines -> route-service, /admin/drivers -> user-service)
- Route /api/v1/public/** -> Public aggregator (Likely points to trip-service or a specific BFF).
